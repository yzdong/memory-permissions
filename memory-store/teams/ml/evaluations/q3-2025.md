# Q3 2025 Eval Results

_Period: July 1 – September 30, 2025_

## Model Evaluated

- Primary checkpoint: `rec-v3.1`
- Holdout cohort: `hs-2025-07`
- Harness version: `v1.8.5`

## Results

```
Precision@10:  0.78
Recall@10:     0.60
NDCG@10:       0.70
MRR:           0.54
p50 latency:   36ms
p99 latency:   135ms
```

Small improvement across the board vs Q2 2025 (`q2-2025.md`). The main contributor was better handling of seasonal signals (summer content trends) added to the feature set in July.

## Slice Analysis

| Slice | Precision@10 | Notes |
|-------|-------------|-------|
| Power users | 0.84 | Strong |
| Casual users | 0.77 | On target |
| New users (cold-start) | 0.59 | Still a gap |
| Mobile traffic | 0.76 | Slightly behind desktop |

Cold-start gap persists. We flagged this in Q2 as well. An experiment with popularity-blend fallback is scheduled for Q4.

## Offline/Online Delta

Delta was within normal range all quarter. See `offline-vs-online-delta.md` for the delta table; Q3 entry shows +5.9% online CTR lift correlating with the 0.78 offline Precision.

## Incidents

- None this quarter affecting eval integrity.
- One data pipeline hiccup on Aug 3 caused a 6-hour gap in feature logging; affected users were excluded from the holdout cohort retroactively.

## Q4 Outlook

- Targeting improved cold-start performance via popularity-blend experiment.
- Harness upgrade to v2.0 is on the Q4 roadmap (`eval-harness-readme.md`).
- Considering adding Recall@20 as a tracked metric following notes in `recall-at-k-notes.md`.
