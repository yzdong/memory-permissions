# Q2 2025 Evaluation Results

This document captures the evaluation outcomes for the recommender model over Q2 2025 (April–June).

## Context

Q2 was the first full quarter post-replatform onto the new training infra. The model architecture was unchanged (two-tower retrieval + pointwise ranker), but the training data pipeline was entirely rewritten. We expected some volatility.

## Metrics

### Retrieval Model
- Recall@100: 0.76 (target was 0.74) ✅
- MRR@10: 0.44 ✅

### Ranking Model
- Precision@10: 0.78 (target was 0.77) ✅
- NDCG@20: 0.63 ✅

### Latency
- Median latency: 48ms
- P99 latency: 142ms — slightly over the 130ms target, tracked in issue #4821

## Observations

1. New pipeline produces cleaner deduplication — probably explains the recall improvement
2. P99 latency regression was traced to a serialization bottleneck in the feature fetch layer, fixed in late June
3. The eval harness caught a data leakage bug in May that would have inflated precision by ~3 points — important catch, see `eval-harness-readme.md` for what was added
